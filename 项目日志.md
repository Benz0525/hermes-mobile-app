### 修了三个问题

**1. 首页版本号 v3.3 → v4.1**
- 根因：`ChatScreen.js:424` 硬编码 `v3.3`，从没改过
- 修复：改成 `v4.1`

**2. 升级死循环**
- 根因：前端 `APP_VERSION_CODE=8` 后端 `version_code=9` → 装完还弹
- 修复：前后端统一到版本号对应关系
- **铁律**：发版必须同步三处版本号——`update.js` 的 `APP_VERSION_CODE`、`server.py` 的 `version_code`、`ChatScreen.js` 的显示版本

**3. OTA 升级（最折腾）**

| 尝试 | 方案 | 结果 |
|------|------|------|
| 1 | expo-file-system + expo-intent-launcher | ❌ native .so 没打进APK |
| 2 | react-native-blob-util | ❌ 编译20分钟+，同样native问题 |
| 3 | **intent:// 唤系统下载器** | 🟡 待验证，零原生模块依赖 |

- 核心教训：Expo SDK56 + Hermes引擎 + GitHub Actions bare workflow → 原生模块极不可靠
- 解决思路：用 Android 内置的 `intent://` 协议直接唤起系统 DownloadManager，不依赖任何第三方原生模块

### 4. 输入栏不贴底
- 根因：键盘弹起时 `emptyFull`（flex:1 占位元素）被 `null` 掉，输入栏飘到顶部
- 修复：`emptyFull` 始终渲染，键盘弹起只隐藏内容不隐藏容器

### 5. nginx APK MIME 类型
- 加了 `application/vnd.android.package-archive` MIME 类型
- 加了 `Content-Disposition: attachment` 头
- 路径：`/etc/nginx/sites-enabled/hermes`
